Supply Chain Performance Analytics Project Report

1. Project Overview

The Supply Chain Performance Analytics project was developed to analyze and optimize supply chain operations using Python, SQL, Excel, and Power BI. The project focuses on evaluating operational performance, identifying business bottlenecks, improving inventory management, monitoring supplier performance, and supporting data-driven decision-making.

The analysis was performed on a supply chain dataset containing order transactions, supplier information, warehouse details, product information, and inventory records.

---

2. Business Problem

Modern supply chains generate large volumes of operational data. Organizations often face challenges such as:

- Delivery delays
- Inventory shortages
- Overstock situations
- High shipping costs
- Supplier performance inconsistencies
- Inefficient warehouse utilization
- Lack of KPI monitoring

Without proper analytics, these issues can negatively impact profitability, customer satisfaction, and operational efficiency.

3. Project Objectives

The primary objectives of this project are:

6. Data Cleaning Process

Several preprocessing activities were performed:

Missing Value Analysis

Missing values were identified and reviewed.

Duplicate Removal

Duplicate records were removed to ensure data accuracy.

Data Type Conversion

Date fields were converted into proper datetime format.

Revenue Calculation

Revenue was calculated using:

Revenue = Quantity × Unit Price

Delivery Performance Calculation

Delivery lead time was calculated using:

Delivery Time = Delivery Date − Order Date
